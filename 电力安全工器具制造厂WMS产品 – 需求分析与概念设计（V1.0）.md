# 电力安全工器具制造厂WMS产品 – 需求分析与概念设计（V1.0）



## 1. 业务场景定义

- **企业类型**：苏州地区电力安全工器具制造厂
- **核心产品**：绝缘棒、接地线、绝缘手套、验电器、绝缘梯、安全带等
- **业务痛点**：
  - 工器具具有强制预防性试验周期（如绝缘手套6个月），超期使用有安全风险
  - 来料和成品需严格质检，不合格品必须隔离
  - 车间高频借用工器具，需记录借还、到期提醒
- **WMS目标**：实现原材料、半成品、成品的全流程库存管理，通过智能工器具柜，使一线工人能够自助完成借用和归还，且该操作能即时反馈到库存状态中



## 2. 用户角色与核心诉求

| 角色          | 职责                           | 核心诉求                                 |
| ------------- | ------------------------------ | ---------------------------------------- |
| 厂长/高层     | 查看库存报表、周转率           | 数据可视化、异常预警                     |
| 仓库主管      | 管理仓库人员、盘点、处理异常   | 库存准确、超期预警、流程顺畅             |
| 质检员        | 来料/成品检验，判定合格/不合格 | 待检任务清晰、快速录入结果、隔离不合格品 |
| 仓管员        | 收货、上架、拣货、发货         | 简单操作、减少差错、实时同步库存         |
| 生产线/领料员 | 生产领料、半成品交接           | 快速领料、不耽误生产                     |
| 销售/发货员   | 销售出库、物流安排             | 库存可承诺量透明、发货及时               |
| 一线工人      | 借用/归还工器具                | 自助操作、超期提醒、不影响作业           |

## 4. 产品与物料范围

### 3.1 成品主要类别
- 基本绝缘：验电器、绝缘杆、接地线、绝缘隔板
- 辅助绝缘：绝缘手套、绝缘靴、绝缘胶垫
- 个体防护：安全帽、安全带、防电弧服
- 登高工具：绝缘梯、脚扣、防坠器
- 警示标识：安全围栏、标志牌

### 3.2 原材料主要类别
- 绝缘材料：玻璃纤维布、环氧树脂、硅橡胶
- 金属材料：无氧铜丝、合金钢、铝合金
- 化工材料：固化剂、涂料
- 纺织材料：高强度聚酯纤维、尼龙
- 配件与包材：工程塑料、包装箱

| 待检/隔离区 | 来料待检、成品待检、不合格品、报废品             | 所有物料先入此区，检验合格后方可移出；不合格品严格隔离       |
| 原材料仓    | 绝缘材料（恒温恒湿）、金属、化工（防爆）、包材   | 根据物料特性分区存放                                         |
| 半成品仓    | 已焊接台架、注塑件、裁切毛坯                     | 生产线在制品暂存                                             |
| 成品仓      | 绝缘棒、接地线、绝缘手套等合格成品               | 绝缘棒、接地线、绝缘手套等合格成品。每一件成品都标注序列号和下次试验日期（手写标签或贴纸），超期品会被移出成品仓或标记禁用 |
| 车间借还点  | 从成品仓调拨的高频借用物品（如绝缘手套、验电器） | 位于车间，由仓管员人工管理借还，使用纸质借据                 |



## 6. 关键业务规则


1. **待检/隔离区强制规则：** 任何物料（原材料或成品）必须先放置在待检区，经过质检判定合格后，才能移入相应的合格库区（原材料仓/半成品仓/成品仓）。

2. **有效期管理规则：** 绝缘类成品（如绝缘手套、验电器）必须标注下次预防性试验日期。超过试验日期的工器具，禁止出库、禁止借给工人使用。当前由仓管员人工核对标签或台账来执行。

3. **不合格品隔离规则：** 一旦判定为不合格，物料必须存放在隔离区，严禁与合格品混放。不合格品不得用于生产、不得出库、不得借用。当前由质检员贴标签、仓管员人工区分位置。

4. **库存移动记录规则：** 每一次物料的入库、出库、借出、归还、报废，都必须在手工台账上记录，以保证库存账目可追溯。

5. **唯一标识规则：** 每一件绝缘成品（如绝缘棒、绝缘手套）应有唯一的产品编号或序列号，用于区分批次和追溯有效期。当前由生产时贴标签或手写编号。

6. **借用归还规则：** 工人借用工器具必须填写借据（含借用人、工具名称、数量、借用时间、预计归还时间），归还时仓管员需检查完好并登记归还时间。超期未还的，由仓管员负责催收。



## 7. 核心流程概述


### 7.1 采购入库流程
1. 采购到货，仓管员将原材料放置待检区 → 仓管员通知质检员检验到货原料
2. 质检员抽样检查 →将不合格品贴上不合格标签
3. 仓管员将检验合格的批次原料入库原材料仓

**常见异常及当前处理**：
- 仓管员将不合格品放置在隔离区 → 仓管员通知采购员将不合格原料退货或者换货 → 采购员安排退货或者换货

### 7.2 生产入库流程
1. 生产完工的半成品和成品，需要质检员进行检验 → 不合格品贴上不合格标签
2. 检验合格的半成品和成品，分别入库至半成品仓和成品仓

**常见异常及当前处理**：
- 不合格的半成品和成品 → 存放在隔离区 → 由仓管员通知产线工人返工或者报废 → 产线工人进行返工或者报废处理


### 7.3 生产领料流程
1. 生产计划员下达生产工单（纸质） → 领料员持工单到原材料仓 
2. 仓管员根据工单拣货 → 交付领料员，领料员确认签字 → 领料员领料出库


### 7.4 成品出库流程
1. 销售内勤下达销售订单（纸质或口头）
2. 仓管员到成品仓查找对应成品 → 手工核对有效期（翻阅台账） → 拣货 
3.  复核人员再次核对产品、数量、有效期 → 发货（手工填写出库单） 
4.  仓管员扣减手工台账


### 7.5 工器具借还流程（车间借还点，人工管理）
1. 工人手写借据（工器具名称、数量、借出时间、预计归还时间），交给仓管员
2. 仓管员确认工器具在车间借还点且未过期 → 借出给工人，并在手工库存台账上备注“已借出”
3. 工人归还时，仓管员检查完好后放回原处，并在台账上备注“已归还”

**常见异常及当前处理**：
-  工人超期未还 → 仓管员电话或当面催收
-  所需工器具不在车间借还点 → 仓管员告知工人等待或从成品仓临时调拨

## 11.主要原型页面说明



### 1.页面名称：收料工作台

**页面路径**：入库管理 → 收料工作台

**核心功能**：仓管员查看已到货批次，执行确认收料操作，启动质检流程。

**核心逻辑**：

- 收料工作台的数据由系统自动生成，非手动创建。货物到达时，仓管员用PDA扫描采购单号，录入到货数量，系统自动生成一条收料记录
- 到货数量由仓管员在PDA扫码时录入（实际清点后的数量），采购单号由系统自动关联
- PC端收料工作台仅用于查看和“确认收料”，数据源头在PDA端
- 点击“确认收料”后，状态变为“待入库”，系统自动触发质检任务推送至质检员
- 同一采购单可分多次到货，每次PDA扫码生成独立的收料记录

**列表字段**：采购单号、供应商、物料编码、物料名称、规格型号、到货数量、到货时间、状态、操作

**状态说明**：

- 待确认：PDA已扫码生成记录，等待PC端确认。可执行“确认收料”和“查看”
- 待入库：已确认收料，等待质检完成后入库。不可操作，仅查看
- 已入库：已完成入库，流程终结。不可操作，仅查看
- 已退货：已退回供应商，流程终结。不可操作，仅查看

**页面规则**：

- 页面加载时默认选中“待确认”状态，仅显示待确认批次
- 支持按采购单号/物料名称模糊搜索，按供应商、到货时间区间、状态筛选
- 仅“待确认”状态显示“确认收料”按钮
- 点击“确认收料”弹出二次确认弹窗，确认后状态变为“待入库”，该行从待确认列表移除，系统自动触发质检任务
- 点击“查看”按钮跳转至收料详情页
- 支持批量确认（仅待确认批次可勾选）
- 支持导出当前筛选条件下的收料数据（Excel格式）
- 默认按到货时间降序排列

**异常与边界**：

- 列表为空时，显示“暂无待确认收料批次”
- 搜索结果为空时，显示“未找到匹配的收料记录”
- 页面加载中时，显示加载状态
- 确认收料操作失败时，提示“操作失败，请稍后重试”，状态不变

**权限说明**：

- 仓管员：可查看、确认收料、导出
- 仓库主管：可查看（只读，不可确认收料和导出）
- 厂长/高层：可查看（只读，不可确认收料和导出）



### 2.页面名称：入库单列表

**页面路径**：入库管理 → 入库单列表

**核心功能**：展示所有已完成上架入库的物料单据，供仓管员/主管追溯入库记录、核对库存来源。待入库和入库中的任务由PDA端处理，不在此页面展示

**核心逻辑**：

- 入库单列表仅展示状态为“已完成”的入库单据（待入库/入库中的任务在PDA端处理，PC端不展示）
- 数据由系统自动生成，非手动创建。仓管员用PDA完成所有物料上架确认后，系统自动生成入库记录并写入该列表
- 1个入库单号可包含多行物料明细（同来源单号、同仓库、不同物料编码、不同库位）。入库单通过“仓库”字段定位所属仓库，通过“库位”字段定位每行物料的具体位置，不支持多仓库合并在一张入库单中（原材料、半成品、成品分属不同仓库，按实际仓库分别生成入库单）

**列表字段**：入库单号、物料类型、来源单号、物料编码、物料名称、规格型号、入库数量、仓库、库位、入库时间、操作

**页面规则**：

- 支持按（入库、来源）单号/物料名称模糊搜索，按物料类型、入库时间区间筛选
- 点击“新建入库单”弹出新建入库单弹窗（用于无来源单号需手动入库的场景），提交后关闭弹窗并刷新列表
- 点击“查看”按钮跳转至入库单详情页
- 点击“打印”按钮跳转至当前入库单详情的打印页面
- 支持导出当前筛选条件下的入库单数据（Excel格式）
- 默认按入库单号降序排列

**异常与边界**：

- 列表为空时，显示“暂无数据”
- 搜索结果为空时，显示“未找到匹配的入库单数据”
- 页面加载中时，显示加载状态

**权限说明**：

- 仓管员/仓库主管：可查看、新建、导出、打印
- 厂长/高层：可查看、打印（不可新建和导出）



### 3.页面名称：拣货工作台

**页面路径**：出库管理 → 拣货工作台

**核心功能**：仓管员查看待拣货明细，执行拣货操作，完成出库前的拣货作业

**核心逻辑**：

- 拣货工作台的数据由系统自动生成，非手动创建。ERP/MES发出出库指令后，待出库单审核通过后，系统自动生成拣货单，展示拣货明细，等待仓管员执行出库前的拣货作业
- 1个来源单号（销售订单/生产工单）可对应多个拣货单号（分批拣货场景），1个拣货单号只对应1个来源单号。1个拣货单号可包含多个物料编码、多个推荐库位
- PC端拣货工作台仅用于查看和“去拣货”（状态切换），拣货作业由PDA端执行
- 点击“去拣货”后，拣货单状态由“待拣货”变为“拣货中”，并推送至PDA端仓管员的任务列表；PDA端完成拣货后，PC端状态自动切换为“已完成”

### 10.页面名称：借还管理

**核心功能**：仓库主管/仓管员查看所有智能柜的借还记录，执行催还操作

**核心逻辑**：

- 此页面仅仓库主管/仓管员可见并操作，展示所有智能柜的借还单据，不限于当前登录者
- 借还单号由借料申请被审核通过后，系统自动生成，并自动关联对应的申请单号，二者是1对1的关系。1个借还单号对应1个申请单号，对应1个物料编码和1个借用人
- 借还单中的物料信息、借用人信息均从关联的申请单继承，借还单本身不产生新数据
- 实际借还执行动作（取走/归还）通过智能柜完成，借还单在系统中自动同步状态
- 催还包含系统自动催还和人工催还两条独立路径：系统自动催还由后台定时任务触发（归还日期前1天起每日发送提醒，超期后强度递增，直至工器具归还）；人工催还由本页面“催还”按钮触发，不依赖系统自动催还的时间规则

**列表字段**：借还单号、关联申请单号、物料编码、物料名称、规格型号、数量、借用人、借用时间、应还时间、实际归还时间、状态、操作

**状态说明：**

- 借出中：工器具已被工人取走，正在使用中，尚未归还。可执行“催还”和“查看”
- 超期未还：已超过应还时间，工器具仍未归还。可执行“催还”和“查看”
- 已归还：工器具已归还，流程完结。可执行“查看”

**页面规则**：

- 页面加载时默认按借还单号降序排列（最新单号排最前）
- 支持按（借还、关联申请）单号/物料名称/借用人模糊搜索，按状态、借用时间区间筛选
- 点击“催还”后，系统立即向借用人发送催还通知（短信/APP/企业微信），并记录催还日志（操作人、操作时间、催还类型为“人工催还”）。催还按钮在借出中和超期未还状态下始终显示，不限制催还时间
- 系统自动催还与人工催还相互独立：系统自动催还由后台定时任务触发，不依赖本页面操作；人工催还由本页面“催还”按钮触发，不依赖系统自动催还的时间规则。两者各自独立运行，互不干扰
- 点击“查看”跳转至借还单详情页
- 支持导出当前筛选数据（Excel格式）

**异常与边界**：

- 列表为空时，显示“暂无数据”
- 搜索结果为空时，显示“未找到匹配的借还记录”

- 催还通知发送失败时，提示“催还通知发送失败，请稍后重试”，借还单状态不变

**权限说明**：

- 仓库主管：查看、催还、导出
- 仓管员：查看、催还、导出（仅限自己管辖区域的记录）

### 12. PDA扫码执行页（仓管员）

**页面路径**：PDA端 → 我的任务→扫码执行（仓管员）

**核心功能**：仓管员通过连续扫码完成入库上架、拣货、盘点任务的具体执行操作

**顶部导航**：左侧返回按钮（返回任务列表），居中显示“扫码拣货/扫码入库/扫码盘点”（根据任务类型动态变化），右侧消息提醒图标（点击查看系统通知）。

**核心逻辑**：

- 扫码框始终存在并保持聚焦，支持连续扫码
- 每次扫码成功后，数量+1，进度条同步推进
- 物料清单中已完成的行自动标记为完成样式，无需手动确认
- 全部物料完成后，“完成提交”按钮变为可点击

**页面元素**：顶部任务信息（任务类型、进度条、来源单号）、中部物料清单（每行前方显示状态标签：待处理/处理中/已完成，展示物料编码、名称、已处理/待处理数量）、扫码框（位于清单上方）、底部操作按钮（异常上报、暂停、完成提交）

**交互规则**：

- 扫码后数量自动累加，无需额外操作
- 同一物料多次扫码，已处理数量持续累加
- 全部扫码完成后，点击“完成提交”返回任务列表
- 支持手动输入数量

**异常与边界**：

- 扫描到不属于当前任务的物料时，提示“该物料不属于当前任务”
- 单个物料超出需求数量时，提示“已超出需求数量，请确认”
- 全部物料完成后，进度条100%，“完成提交”可点击

**权限说明**：

- 仓管员：执行（仅限分配给自己的任务）

### 14. PDA扫码质检执行页（质检员）

**页面路径**：PDA端 → 我的任务→扫码质检执行（质检员）

**核心功能**：质检员通过扫码逐件检验物料，对每件物料做出“合格”或“不合格”判定

**顶部导航**：左侧返回按钮（返回任务列表），居中显示“扫码质检”，右侧消息提醒图标

**核心逻辑**：

- 扫码前只显示扫码框和物料清单，不显示“合格/不合格”按钮
- 扫码成功后，判定按钮出现，扫码框被覆盖，必须完成判定才能继续
- 点击“合格”后，已检数量+1，判定按钮消失，扫码框恢复
- 点击“不合格”后，弹出原因选择面板，选择原因确认后，物料标记为不合格，已检数量+1，不合格数量+1，判定按钮消失，扫码框恢复
- 所有物料判定完成后，“完成提交”按钮变为可点击

**页面元素**：顶部任务信息（任务类型、进度条、质检单号）、中部物料清单（每行前方显示状态标签：待处理/处理中/已完成，展示物料编码、名称、待检数量、已检数量、不合格数量）、扫码框（位于清单上方）、判定按钮（扫码后覆盖在扫码框上）、底部操作按钮（异常上报、暂停、完成提交）

**交互规则**：

- 每次扫码后必须完成判定才能继续
- 不合格时需选择原因，必选
- 全部判定完成后，点击“完成提交”返回任务列表

**异常与边界**：

- 不合格时弹出原因选择面板，选择原因后确认，记录不合格原因
- 已判定完成的物料再次扫码时，提示“该物料已判定完成”
- 全部判定完成后，进度条100%，“完成提交”可点击

**权限说明**：

- 质检员：执行（仅限分配给自己的任务）
